# 設備序號掃描與登錄手機 Web App (Mobile Device Serial Number Scanner)

這是一個專為手機與行動瀏覽器設計的**設備序號登錄與條碼盤點 App**。使用者可以直接開啟相機快速掃描設備條碼，系統會自動在畫面上顯示條碼清單，並在結束後一鍵匯出為純序號文字檔 (TXT) 或單一欄位的 CSV 表格。

---

## 🌟 核心特色
1. **版面高度優化**：移除了頂部佔空間的統計欄，將 **「已登錄數量」** 計數器直接整合至已登錄清單標題旁，大幅拉長手機畫面中的清單顯示區域（最高可瀏覽 `480px` 高度），適合大量盤點。
2. **純條碼內容清單**：列表卡片精簡為**僅顯示設備序號與刪除垃圾桶 `🗑️`**，無其他多餘資訊（如日期、備註等），極速瀏覽與整理。
3. **3 秒防重複掃描冷卻**：
   - 每次成功識別條碼後，系統會**暫停掃描 3 秒鐘**，提供充足時間移開鏡頭。
   - 期間雷射掃描光束會自動**變為灰色並暫停動畫**，並提示「已暫停掃描，請移開鏡頭...」。
4. **重複序號警告**：每個序號限登錄一次。若掃描或輸入已登錄的序號，會觸發**警示音效**（低音雙嗶聲）與**雙重震動**，彈出警告視窗防呆，點擊關閉即可重新掃描。
5. **純淨數據匯出 (雙格式可選)**：
   - **TXT 檔案**：每行一個設備序號，無其他編號、字樣或格式，非常適合直接全選複製並導入您的 ERP 或其他系統。
   - **CSV 檔案**：單一欄位「設備序號」表格，內建 UTF-8 BOM 避免 Excel 中文亂碼。
6. **使用中鏡頭顯示與手電筒防呆**：
   - 頂部狀態列會同步顯示當前 **「使用中鏡頭：[鏡頭名稱]」**。
   - 手電筒功能僅在切換至**後置相機 (Back Camera)** 時才可點選，前置鏡頭時會自動鎖定並轉灰。
7. **零依賴與動態音效**：使用 Web Audio API 自動合成嗶嗶聲與警告音，不需下載額外 MP3，載入速度極快。

## 🚀 啟動與測試步驟

因為瀏覽器安全策略規定，**必須在 `localhost` 或 `https://` 安全連線下，才允許網頁調用相機（`getUserMedia`）**。直接用瀏覽器開 `file:///index.html` 將無法開啟相機。

我們為此專案附帶了一個**零依賴的本地伺服器腳本**，請按以下步驟啟動：

### 在 Windows 電腦上啟動：
1. 以滑鼠右鍵點擊 [run_server.ps1](file:///d:/Antigravity/inventory-scanner/run_server.ps1)，選擇 **「以 PowerShell 執行」**（Run with PowerShell）。
2. 系統會自動啟動本地伺服器，並打開瀏覽器前往 `http://localhost:8080`。
3. 即可在網頁上看到相機視窗並開始測試！

### 📱 用手機連接掃描（真正的手機操作體驗）：
1. 確保您的電腦與手機連接到**同一個 Wi-Fi 網域**。
2. 啟動 `run_server.ps1` 後，電腦終端機會列出您的區域網路 IP。
3. 用手機瀏覽器（如 Chrome, Safari）輸入網址：`http://192.168.0.157:8080`。
4. 允許瀏覽器調用相機權限（Android 設備請按照畫面上提示設定 Chrome Flags 解鎖連線），即可開始用手機相機掃描實體商品條碼！
